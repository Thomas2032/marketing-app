5. 前端架構與三大核心功能

詳解 Next.js 16 狀態驅動機制，以及專案工作區、視覺編輯器、發佈精靈的實現原理。

1. 前端資料流與異步輪詢機制 (State & Polling)

前端依靠 `USE_BACKEND` 開關控制資料層。設為 `true` 時，前端向 FastAPI 發送 HTTP 請求；設為 `false` 時，前端加載 `lib/mock-campaign.ts` 及 `lib/mock-orchestrator.ts`，在 `sessionStorage` 內模擬異步運行，支援無後端環境的展示。

遞歸輪詢生命週期 (Polling Loop Lifecycle)
[CampaignResults.tsx] ➜ useEffect 偵測 status == "running"
➔ 呼叫 poll() ➔ 發起 API 請求 GET /campaigns/{id}
➔ 收到後端 response ➔ 更新 Campaign React State
➔ 判斷最新 status 是否屬於終端狀態 (TERMINAL_STATUSES: "completed", "failed", "review")？
➔ 若「是」：終止輪詢，退出 poll()
➔ 若「否」：排程 setTimeout(poll, 3000) 於三秒後重新觸發

2. 專案工作區管理 (Project Workspace)

專案工作區提供企劃案的分組管理功能，對應 `types/project.ts` 中的 `Project` 結構，實現了多專案資料夾切換。

持久化機制 (Storage)

目前採用客戶端暫存 `sessionStorage` (`mock-projects`) 持久化專案資訊，以 `active-project-id` 追蹤當前啟用的專案 UUID。在動態路由 `/projects/[id]` 載入時，前端自動從 sessionStorage 撈取關聯的 Campaign 清單並渲染至工作區。

主要組件 (Components)
  • ProjectSidebar：展示專案資料夾清單，支持刪除與修改名稱。
  • CreateProjectForm：防呆與防空專案名稱輸入的建立表單。
  • ProjectCampaignList：顯示屬於當前動態 `projectId` 的行銷企劃案。

3. 視覺圖像編輯器 (Visual Image Editor)

視覺編輯器讓用戶可以在 AI 生成的底圖上進行圖層疊加，或框選特定區域發起 AI 二次編修（AI Inpainting/Editing）。

坐標歸一化幾何計算 (Relative Coordinate Scaling)

為了解決不同瀏覽器視窗與螢幕解析度下，標註圖層與選取框位置跑掉的問題，編輯器實現了**歸一化坐標系 (Normalized Bounding Coordinates)**：

// 坐標計算公式 (clientX ➜ 0~1 相對坐標)
const rect = container.getBoundingClientRect();
const relativeX = (clientX - rect.left) / rect.width;
const relativeY = (clientY - rect.top) / rect.height;
const relativeW = width / rect.width;
const relativeH = height / rect.height;

當標籤載入時，乘上容器的即時寬高像素進行定位：`left: relativeX * containerWidth`，確保框選元素自適應縮放。

圖層類型與操作模式
1. AI 局部框選圖層 (AI Selection) 以滑鼠 Drag-to-select 框出選區，並能在右側面板為該框選區撰寫 Prompt 指令（如：「把跑車換成紅色」），準備發起 AI 修復。
2. 文字註解圖層 (Text Annotations) 雙擊插入 `contentEditable` 文本框。選取時會彈出控制工具，支持調整字體大小 (Font Size: 12px~48px) 與文字顏色。

4. 批次行銷發佈精靈 (Publish Wizard)

發佈精靈引導用戶從多個 Campaigns 中提取 copywriting 出力，並經過平台排版字數校驗後，批次排程或排發至對接平台。

精靈步驟工作流 (Stepper Steps & Validation)
Step 1
撈取與選取 (Pick Copy)

調用 copy-history.ts，掃描專案內的所有 outputs 產出，按社群平台（X, LinkedIn, Instagram, Facebook）過濾，加入發佈隊列。

Step 2
字數長度校驗 (Validate Limits)

加載 platform-limits.ts 進行即時字數檢查：

X (Twitter): Max 280 字符
Instagram: Max 2200 字符
LinkedIn: Max 3000 字符
Facebook: Max 63206 字符
※ 超過字數限制時，按鈕會 Disabled，防止送出錯誤格式。
Step 3
排程與確認 (Publish/Schedule)

設定發佈時間（`datetime-local`），模擬 API 發送批次，並在成功頁面顯示各平台 API 回傳狀態。
