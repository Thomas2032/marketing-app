1. System Overview

High-level understanding of the codebase structure, execution models, and technology stack.

FE

Next.js 16 Frontend

Responsible for displaying the campaigns catalog, interactive creation forms, and rendering real-time sub-agent runs via a recursive polling loop. Includes offline mock mode simulation.

API

FastAPI API Layer

Validates payloads, manages database sessions, triggers background tasks or queues via Redis, and serves campaign state increments to frontend clients.

AG

LangGraph Agent Loop

Executes the campaign generation workflow. Combines individual sub-agent nodes (extract, write, generate image, review) and persists state progress incrementally to the DB.

Core Design Principles

Agent-as-a-Tool

One Orchestrator handles every user turn. Specialist capabilities (writing, image generation, extraction, review) are exposed as tools the orchestrator calls dynamically rather than a rigid static pipeline.

Continuous Conversation

Follow-up chat messages append to the campaign message history and trigger a rerun of the orchestrator. Nodes dynamically digest prior chat history to modify outputs.

Incremental State Commits

Instead of committing database updates only at the end of a long-running graph, nodes commit status updates and outputs mid-run. This enables immediate feedback in the UI.

Optimistic UI & Polling

The frontend performs optimistic updates on messages and starts a 3000ms recursive polling loop to fetch incremental updates, ensuring a fluid user experience.

Quick Start & Execution Guide

Follow these commands to configure the development stack locally.

# 1. Spin up PostgreSQL and Redis Infrastructure

docker compose up -d postgres redis

# 2. Configure and Run Backend (FastAPI on Port 8000)

cd backend && cp .env.example .env # (Optional) Edit .env to add OPENAI_API_KEY and AWS/Cloudflare storage keys python3 -m venv .venv && source .venv/bin/activate pip install -r requirements.txt uvicorn app.main:app --reload --port 8000

# 3. Configure and Run Frontend (Next.js on Port 3000)

cd frontend && cp ../.env.example .env.local # Set NEXT_PUBLIC_USE_BACKEND=true (Default: true) to bypass mock mode npm install npm run dev
