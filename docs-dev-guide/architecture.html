2. Architecture & LangGraph Agent Loop

Linear execution pipeline driven by StateGraph nodes and intermediate database updates.

The campaign generation process is modeled as a stateful graph powered by LangGraph. The execution loop transitions data through discrete specialized agent nodes. Intermediate state and generated outputs are committed to the database incrementally at the start and completion of each node execution, rather than only at the end of the entire graph run. This enables the frontend to poll and render step-by-step progress bars in real-time.

Extractor Node (`extractor_node`)

Parses the initial campaign brief into structured fields.

output_type: "extracted"
State Input Requirements:
  • brief (Original text user prompt)
State Modifications:
  • extracted (Object with: product, audience, tone, key_messages, constraints)
Database Action:

Invokes db_start_tool_call with "extract_brief". Upon parsing successfully, stores the JSON metadata in a CampaignOutput (type: "extracted"), maps the FK output_ref in state.tool_calls, and updates the state.extracted column.

backend/app/agents/nodes/extractor.py Python 
async def extractor_node(state: CampaignGraphState) -> dict:
    campaign_id = state["campaign_id"]
    tool_run_id = await db_start_tool_call(campaign_id, "extract_brief")
    try:
        settings = get_settings()
        llm = ChatOpenAI(model=settings.openai_model, api_key=settings.openai_api_key, temperature=0.2)
        response = await llm.ainvoke(
            [
                SystemMessage(content=EXTRACTOR_SYSTEM),
                HumanMessage(content=state["brief"]),
            ]
        )
        content = response.content
        parsed: ExtractedBrief = json.loads(str(content))
        await db_complete_tool_call(
            campaign_id=campaign_id,
            tool_run_id=tool_run_id,
            tool_name="extract_brief",
            output_type="extracted",
            content=None,
            metadata_=parsed
        )
        return {"extracted": parsed}
    except Exception:
        await db_fail_tool_call(campaign_id, tool_run_id, "extract_brief")
        raise

Incremental State Commit Pattern

To enable asynchronous progress tracking, each node relies on transactional commits mid-run using app/services/campaign_state_helper.py. This module runs queries inside discrete DB sessions to bypass the main execution thread's uncommitted transaction.

1. Start of Tool Run (Pending Indicator) 
# Incremental db_start_tool_call
# Inserts a 'running' entry in state.tool_calls
# and commits immediately.
tool_calls.append({
    "tool_run_id": f"tool-{uuid.uuid4()}",
    "tool": tool_name,
    "status": "running",
    "started_at": datetime.utcnow().isoformat()
})
campaign.state = current_state
await session.commit()
2. Tool Completion (Payload + Output Link) 
# Incremental db_complete_tool_call
# Upserts CampaignOutput (to overwrite old iterations)
# updates tool_calls to 'completed' with output_ref,
# updates campaign.state fields, and commits.
output = CampaignOutput(output_type=output_type, ...)
session.add(output)
await session.flush()

call["status"] = "completed"
call["output_ref"] = str(output.id)
current_state[output_type_key] = content/metadata_
await session.commit()
